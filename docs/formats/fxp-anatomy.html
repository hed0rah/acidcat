A .fxp is a VST 2 single-preset file: a fixed 28-byte CcnK header, an optional 28-byte preset name, then a payload that is either a list of float parameters (FxCk) or an opaque plugin blob (FPCh). All multi-byte fields are big-endian, the classic parsing trap on a little-endian host. Hover any field or byte to light its span and read the decode; click a field with a + to open its table. Colour marks kind (see the key). Example: an Xfer Serum (XfsX) opaque-chunk preset.

▸preset name0x1C . 28 bytes

Single-preset variants (FxCk / FPCh) carry a 28-byte null-padded ASCII program name here. Banks put 128 reserved bytes first, then per-program data.

▸payload0x38 . plugin-specific

For FxCk, num_params float32 values. For FPCh (this file), a uint32 BE chunk_size at 0x38 then that many opaque bytes the plugin owns. Not portable across plugins; acidcat reports it as a region, not decoded.
