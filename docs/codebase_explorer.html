Everything acidcat does is one of three flows: ingest (parse files into a database), analyze (compute what metadata does not say), and query (fan out across every registered library). The diagrams below trace each one through the modules above.

Under the hood: signal analysis

The analysis layer exists because most sample libraries lie. Filenames claim tempos the audio does not have, key tags are missing or wrong, and ACID chunks survive format conversions their data did not. acidcat treats every source as a witness and cross-examines them.

BPM estimation

  • librosa computes an onset strength envelope, then tempo candidates two ways: once from the envelope, once from raw audio. The median of all candidates becomes detected_bpm.
  • In parallel, filename parsing hunts for 140bpm, bpm140, and bare 2-3 digit runs, rejecting digits glued to letters so pack codes like 91V_SBH_126_... resolve to 126, not 91.
  • validate_and_improve_bpm reconciles the two. Beat trackers commonly lock onto half or double time, so it tests 2x, 0.5x, 1.5x, and 1/1.5x corrections against the filename claim before trusting either side.
  • Anything under 256 samples is declared a one-shot and skips tempo entirely.

Key detection, and why acidcat refuses to guess

  • A chroma vector collapses the spectrum into 12 pitch-class bins. Its argmax names the strongest pitch class, but a pitch class is not a key: A minor and A major share a tonic and differ in mode.
  • Downstream code treats a bare letter as major, so emitting "A" for an A minor sample would route find_compatible to the wrong side of the wheel. Since v0.5.5 chroma-only detection returns None and the filename parser, which carries mode explicitly, wins.
  • The honest fix is Krumhansl-Schmuckler key profiles: correlate the chroma vector against 24 rotated major and minor templates and take the best. On the roadmap; refusing to guess beats guessing wrong.

Harmonic compatibility

  • The Camelot wheel renames the circle of fifths into clock positions DJs can add and subtract: 8A is A minor, 8B is its relative C major, and mixing rules become "move one number or flip the letter".
  • parse_key accepts whatever the wild ships: Am, A minor, F#min, Bb maj, Camelot codes like 5A, octave forms like C4, and the capital-M major convention (CM) used by Beatport, Mixed In Key, Serato, and Rekordbox.
  • Enharmonic spellings are expanded on filter (Db and C# both hit), and flats are normalized to sharps at ingest so the DB holds one canonical form.

The 51-dimension feature vector

  • 13 MFCCs (mean and std each) summarize timbre: mel filterbank, log compression, then a DCT that decorrelates the bands. Coefficient 1 tracks overall energy tilt; higher ones capture formant-like texture, which is why they cluster growls near growls.
  • Spectral shape stats: centroid (brightness), rolloff (where 85 percent of energy sits), bandwidth, contrast (peak vs valley per octave band), plus zero-crossing rate and RMS energy for noisiness and loudness.
  • Tonnetz coordinates project chroma onto interlocking circles of fifths and thirds, giving a 6-dim harmonic position where consonant keys sit near each other.
  • Rhythm contributes tempo_librosa and a beat count from beat tracking; chroma and mel spectrogram statistics round out the set.
  • Similarity z-scores every column then ranks by cosine distance, so a quiet and a loud version of the same kick still match on shape.
  • The index also runs kind inference: ACID beats or 2+ seconds means loop, under 1 second with no beats means one-shot.

File format internals

RIFF / WAV

  • The chunk walker seeks chunk to chunk: 4-byte id, uint32 little-endian size, payload, plus a pad byte when the size is odd. Payload reads are capped at 64 KB so a malformed size field cannot allocate gigabytes.
  • The acid chunk is the project's namesake: Sonic Foundry ACID wrote tempo, beat count, meter, and root note into WAVs two decades ago and half the sample industry still does.
  • The smpl chunk carries the sampler's root key and loop points. Root key 0 means "unset", not C-1; a shared helper strips that sentinel at every call site.
  • cue chunks hold marker offsets. The declared count is capped against the actual payload size, since a hostile uint32 here once meant 4 billion loop iterations.

AIFF / AIFC

  • Apple's mirror image of RIFF: FORM container, big-endian everything, COMM instead of fmt.
  • The sample rate is stored as an 80-bit IEEE 754 extended-precision float, a 68k-era format Python's struct cannot unpack. acidcat reassembles it by hand from the 15-bit exponent and 64-bit mantissa.

Standard MIDI Files

  • Delta times and meta lengths are variable-length quantities: 7 bits of payload per byte, high bit set means continue.
  • Running status lets consecutive events omit a repeated status byte. Every mainstream DAW exports this way, and the parser's bookkeeping here is exactly where v0.5.5 fixed a one-byte desynch that corrupted note counts.
  • Tempo lives in FF 51 meta events as microseconds per quarter note; key signature in FF 59 as a count of sharps or flats plus a minor flag.

Serum presets and tagged formats

  • Serum .fxp files open with an XferJson magic header; a JSON metadata block (name, author, tags) precedes the binary wavetable data and is decoded in one linear raw_decode pass.
  • MP3, FLAC, OGG, and M4A go through mutagen with one extractor per tag dialect: ID3v2 frames, Vorbis comments, MP4 atoms. A stray UTF-8 BOM in tag text is stripped before storage.

Storage layer

One SQLite database per registered library, plus one global registry that knows where they all live. Reads fan out; writes route to a single owner via longest-prefix matching.

# central default (no --in-tree) ~/.acidcat/ registry.db # every library: label, root, db_path, stats libraries/ <label>_<hash12>.db # per-library index (12-hex path hash) <label>_<hash12>.db-shm # WAL shared memory <label>_<hash12>.db-wal # WAL log # in-tree opt-in (--in-tree): index travels with the samples <library_root>/.acidcat/ index.db
  • The per-library schema separates immutable audio facts (samples, 21 columns) from user annotations (tags, descriptions), librosa output (features as JSON), scan bookkeeping (scan_roots), and an FTS5 mirror with porter stemming for free-text search.
  • Upserts are INSERT ... ON CONFLICT(path) DO UPDATE, so reindexing is idempotent and mtime+size short-circuits unchanged files.
  • The CLI walker commits every 100 files to amortize fsync; on an HDD-backed sample drive this is the difference between minutes and seconds.
  • register_library refuses nested or duplicate roots inside a BEGIN IMMEDIATE transaction, closing the race between two concurrent registrations.
  • Both DB kinds store a schema_version and refuse to open anything newer than they understand: a loud error instead of silent corruption.
